# 进化计算引擎

## 0. 核心定位

GA 引擎是一个纯计算黑盒。它不读取策略内部字段名，不理解宏观或微观的具体数学含义，只通过抽象接口操作不透明 `Gene`。它的职责是：在固定历史数据与固定出生点上搜索更稳健的参数组合，并把最优个体写入基因库为 `challenger`。

首版只优化纯技术参数，不读取 AI 快照。这样可以保证回测、GA 与晋升流程完全可复现。

## 1. 探索窗口定义

### 1.1 多时段坩埚

首版从 `KLine` 真源表中读取全量历史 `1h` bars，并构造四个窗口：

| 窗口 | 评估区间 | 权重 |
|---|---|---|
| `6m` | 最近 183 天 | 0.10 |
| `2y` | 最近 730 天 | 0.20 |
| `5y` | 最近 1825 天 | 0.30 |
| `full` | 全量历史 | 0.40 |

每个窗口在评估区间之前可带固定 warmup 前缀，warmup 数据只允许用于指标预热，不允许进入评分期。

### 1.2 Gene 操作

策略需实现：

- `Sample`
- `Clamp`
- `Validate`

其中 `Clamp` 负责所有边界修复与结构约束修复。

### 1.3 出生点冻结

`SpawnPoint` 在一个 epoch 内全种群共享且保持冻结。它不参与指纹哈希，不在代内交叉变异。

## 2. 生命周期动作

### 2.1 初始化

默认种群：

- `index 0` 固定为当前种子冠军或默认种子
- 剩余个体按约 `10%` 精英原样、`40%` 精英强化变异、`50%` 完全随机生成

### 2.2 并发评估

评估并发度：

`workers = min(runtime.NumCPU(), populationSize)`

同一个 epoch 内相同基因指纹的个体直接命中缓存，不重复回测。

### 2.3 选择 / 交叉 / 变异

- 选择：`TournamentSize = 3`
- 交叉：均匀交叉
- 变异：各维度独立 Bernoulli 触发，加性高斯扰动

### 2.4 收敛与变异斜坡

若连续若干代最佳适应度改善低于阈值，则按比例提高变异概率与幅度；当变异参数已触顶且仍无改善时，提前终止。

## 3. 适应度函数

首版适应度对每个窗口使用以下定义：

- `Alpha = ROI_strategy - ROI_ghost_dca`
- `SliceScore = Alpha - 1.5 * max(0, MaxDD_strategy - MaxDD_ghost_dca)`

硬否决：

- 若 `MaxDD_strategy >= 0.88`，直接返回 fatal 分数 `-99999`

总分：

`ScoreTotal = 0.10 * 6m + 0.20 * 2y + 0.30 * 5y + 0.40 * full`

首版 Ghost DCA 基准只针对同一 symbol 比较，不做跨资产比较。

## 4. EvolvableStrategy 契约

接口固定包含：

- `StrategyID()`
- `Sample(rng)`
- `Mutate(gene, prob, scale, rng)`
- `Crossover(p1, p2, rng)`
- `Fingerprint(gene)`
- `Evaluate(ctx, gene, plan)`
- `DecodeElite(raw)`
- `EncodeResult(gene, spawn)`

引擎只通过这八个动作操作策略，不读取染色体内部字段。

## 5. EvaluablePlan

`EvaluablePlan` 在 epoch 启动时构建并只读，至少包含：

- `TemplateID`
- `Symbol`
- `SpawnPoint`
- `Windows`
- `DCABaselines`
- `LotStep`
- `LotMin`

首版不把 AI 信号历史快照纳入 `EvaluablePlan`。

## 6. 结果交付与角色

基因角色固定为：

- `challenger`
- `champion`
- `retired`

晋升规则为“硬门槛 + 人工审核”：

1. 四个窗口全部非 fatal
2. `full` 与 `2y` 的 Alpha 必须大于等于 0
3. `full` 最大回撤不得高于 `0.65`
4. 全量复跑必须与原结果完全一致
5. 交易次数达到最小样本量

满足条件后，操作员才能将 `challenger` 晋升为 `champion`。

## 8. 当前明确不做

以下内容不进入 v1 主评分链路：

- AI 信号参与 GA
- 实时调 LLM 参与回测
- 自动晋升 champion
- 多市场统一搜索
- 依赖部分成交或限价挂单模型的复杂回测
