# 策略数学引擎

## 0. 引擎身份

策略引擎是一个纯函数无状态转换器：`Step(StrategyInput, Params) -> StrategyOutput`。它只读取调用方传入的市场快照、账户快照、模板参数与运行时状态，不访问网络、数据库、文件系统或系统时间。

首版仅定义两个模板：`core-btc-v1` 与 `core-eth-v1`。两者的数学结构一致，但参数与基因库完全独立。

## 1. 资产结构三态

系统在策略语义上把标的持仓拆成三类：

- `DeadBTC/DeadAsset`：宏观底仓，只进不出，除非通过显式释放规则转为浮动仓
- `FloatBTC/FloatAsset`：微观浮动仓，可按 `Step()` 决策增减
- `ColdSealedBTC/ColdSealedAsset`：冷封存仓，永不释放、永不卖出

通用公式如下：

- `TotalEquity = USDTBalance + (Dead + Float + ColdSealed) * Price`
- `ReserveFloor = TotalEquity * micro_reserve_pct`
- `SpendableUSDT = max(0, USDTBalance - ReserveFloor)`
- `CurrentMicroWeight = (Float * Price) / max(TotalEquity, epsilon)`

首版默认 `micro_reserve_pct = 0.25`，熊市下可抬高到更保守的有效储备水平。

## 2. 市场状态感知

首版市场状态由可解释规则构成，不使用黑盒状态机。目标不是预测短期涨跌，而是调节宏观资金部署速度与微观引擎激进度。

状态集合：

- `bull`：趋势向上，回撤可控
- `bear`：趋势向下，额外加仓显著减速
- `range`：区间震荡，允许微观层发挥主要作用
- `quiet`：附加标记，表示近端波动很低，微观粉尘订单应尽量归零

规则输入由 `1h` closes 与若干固定窗口统计量组成，例如长短 EMA 相对位置、价格距长均线偏离、近端平均绝对涨跌比率等。首版要求这些规则可解释、可复现、可单元测试。

状态输出至少包含：

- `State`
- `TimeDilationMultiplier`
- `BetaMultiplier`
- `IsQuiet`

其中：

- `bull`：`TimeDilationMultiplier` 可略缩短，`BetaMultiplier` 正常或略高
- `bear`：`TimeDilationMultiplier` 拉长，`BetaMultiplier` 下调
- `range`：两者接近 1
- `quiet`：只用于抑制微观小额订单

## 3. 信号与目标函数

首版不做多因子大杂烩，也不接入 AI 信号。微观层只使用两个技术因子：

- `X_mean_reversion`：价格相对固定 EMA 的标准化偏离
- `X_momentum`：固定窗口对数收益率的标准化值

合成信号：

`Signal = a * X_mean_reversion + b * X_momentum`

其中 `a` 与 `b` 为染色体字段。信号正值偏空、负值偏多。

## 4. 宏观引擎

宏观引擎只负责“长期部署多少资金进底仓”，绝不卖出。它是 DCA 增强层，而不是择时卖出系统。

首版规则固定为：

1. 基础月度 DCA 永远存在，只要 `SpendableUSDT` 与实例资金政策允许，就按月度节奏部署
2. 当价格相对长均线或中长期锚点出现明显折价时，允许触发额外加仓
3. 当资金长期闲置超过死线阈值时，必须有强制部署逻辑，避免长期空仓
4. 在 `bear` 状态下执行“减速但不停”：
   - 保留基础 DCA
   - 折价加仓阈值变严格
   - 单周期额外加仓上限下降
   - 有效储备比例抬高

宏观引擎输出只能是 `BUY` 意图，并且会被 `SpendableUSDT`、最小订单额与策略资金政策约束。

## 5. 微观引擎

微观引擎使用 `Sigmoid 动态天平` 控制浮动仓目标权重：

```text
CurrentWeight  = FloatAsset * Price / TotalEquity
EffectiveBeta  = max(0.01, beta * BetaMultiplier)
InventoryBias  = clamp(CurrentWeight, 0, 1) - 0.5
Exponent       = EffectiveBeta * Signal + gamma * InventoryBias
TargetWeight   = 1 / (1 + exp(Exponent))
```

解释：

- `Signal` 是外力
- `InventoryBias` 是仓位回归力
- `beta` 是外力放大倍数
- `gamma` 决定是否启用仓位回归

随后计算：

- `DeltaWeight = TargetWeight - CurrentWeight`
- `TheoreticalUSD = DeltaWeight * TotalEquity`

再经过楔形区过滤：

1. 若 `|TheoreticalUSD| >= 10.1`，直接下单
2. 若 `|TheoreticalUSD| < 10.1` 且 `IsQuiet = true`，强制归零
3. 若 `|TheoreticalUSD| < 10.1` 且 `IsQuiet = false`，只有当 `|DeltaWeight|` 或 `VolatilityRatio` 超过阈值时，才强制最小订单

在 `bear` 状态下，`BetaMultiplier` 会降低，防止系统在长期下跌中持续把浮动仓越打越满。

## 6. 底仓释放规则

底仓释放只改变 SaaS 账本语义，不直接触发交易所命令。首版保留两种释放：

- `Soft Release`：满足年龄阈值且不属于冷封存仓的底仓，可在“微观层需要更多可卖浮仓”时按比例释放
- `Hard Release`：微观层确有卖出需求且 `Float` 不足时，从非冷封存底仓中补足最小必要数量

冷封存仓在任何情况下都不能被释放。

## 7. 染色体与出生点

染色体只包含会在代内交叉与变异中变化的参数。首版建议字段：

- `signal_weight_mean_rev`
- `signal_weight_momentum`
- `sigma_floor`
- `beta`
- `gamma`
- `discount_buy_threshold`
- `discount_buy_scale`
- `quiet_vol_ratio_threshold`
- `micro_delta_weight_threshold`
- `micro_reserve_pct`

出生点 `SpawnPoint` 不参与交叉和变异，包含：

- `monthly_inject_usdt`
- `idle_deploy_deadline_days`
- `extra_buy_cap_pct`
- `bear_extra_buy_cap_pct`
- `max_release_pct`
- `taker_fee_bps`
- `slippage_bps`
- `global_drawdown_guard`

## 8. 输入输出契约

`StrategyInput` 至少包含：

- `TemplateID`
- `Symbol`
- `Timestamp`
- `Closes`
- `Timestamps`
- `PortfolioSnapshot`
- `RuntimeState`

`StrategyOutput` 至少包含：

- `UpdatedRuntimeState`
- `MacroIntent`
- `MicroIntent`
- `ReleaseIntents`
- `Diagnostics`

首版明确禁止把 AI 信号加入该输入契约。AI 快照只存在于展示链路。

## 9. 适用性声明

本策略只适用于你愿意长期持有的核心资产。它在震荡期与温和趋势期更容易体现微观层价值；在持续单边上涨中可能跑输纯 DCA 或简单持有；在长期单边下跌中，如果标的本身缺乏长期生命力，则系统的心理和数学前提都会失效。

因此首版强制限定标的为 `BTCUSDT` 与 `ETHUSDT`，并在产品层显式禁止山寨币自定义。
