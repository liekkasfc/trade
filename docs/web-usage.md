# QuantSaaS Web 使用说明

本文档面向本地运行的 QuantSaaS Web 界面，说明如何登录、创建实例、查看 Dashboard，以及使用回测、进化和 Data Lab。

## 1. 访问入口

- Web: `http://127.0.0.1:18080/`
- 健康检查: `http://127.0.0.1:18080/healthz`

如果你本地把服务跑在别的地址或端口，请按实际值访问。

## 2. 使用前提

开始前请确认：

- SaaS 后端已经启动
- 当前本地环境可访问配置中的 Postgres 和 Redis
- 当前 `app_role` 为 `dev` 或 `lab`

说明：

- `dev`：工作台、回测、进化、Data Lab 全部可用
- `lab`：只开放回测、进化、Data Lab 和实例只读能力
- `saas`：不开放回测、进化、Data Lab

## 3. 首次使用

### 3.1 先注册账号

当前 Web 页面只有“登录”，没有“注册”按钮，所以第一次使用需要先调用注册接口：

```bash
curl -X POST http://127.0.0.1:18080/api/v1/auth/register \
  -H 'Content-Type: application/json' \
  -d '{"email":"you@example.com","password":"change-me"}'
```

注册成功后，再打开 Web 页面登录。

### 3.2 登录

打开 `http://127.0.0.1:18080/` 后：

1. 确认右上角 `API Base`
2. 输入邮箱和密码
3. 点击 `登录 SaaS`

默认 `API Base` 会读取浏览器本地缓存；如果之前保存过旧地址，手动改回当前 SaaS 地址即可。

## 4. 页面结构

顶部有四个主标签：

- `工作台`
- `回测`
- `进化`
- `Data Lab`

## 5. 工作台

工作台主要负责：

- 查看系统状态
- 登录和清理本地 Token
- 创建实例
- 管理实例
- 查看 Dashboard 和权益曲线
- 查看 LocalAgent 连接状态

### 5.1 状态卡片

页面顶部会显示：

- 引擎状态
- Agent 连接状态
- 总权益
- 实例数量

### 5.2 创建实例

在“快速建实例”区域填写：

- 模板：`core-btc-v1` 或 `core-eth-v1`
- 实例名：自定义
- 资金配额 USDT：例如 `10000`
- 月度 DCA USDT：例如 `500`
- 冷封存资产：例如 `0`
- 最大回撤警戒：例如 `0.35`

点击 `创建实例` 即可。

说明：

- `core-btc-v1` 固定对应 `BTCUSDT`
- `core-eth-v1` 固定对应 `ETHUSDT`
- v1 不支持自由输入其他交易对

### 5.3 管理实例

创建完成后，实例会出现在：

- 左侧 Dashboard 实例列表
- 页面下方“实例工作台”

你可以对实例执行：

- `启动`
- `停止`
- `删除`

### 5.4 Dashboard

选择一个实例后，可以看到：

- 总权益
- 可用资金
- 长期底仓
- 浮动仓位
- 7 / 30 / 90 天权益曲线
- 创建时间
- 首次决策时间
- 决策次数
- 快照数量
- 本月成交数
- 累计成交数
- 最后更新时间

## 6. LocalAgent 状态

工作台内有 `LocalAgent` 状态卡片，会显示：

- 是否在线
- 版本
- 最近心跳时间
- 首次连接时间

如果 Agent 尚未启动，这里会显示离线。  
这不会影响你先使用工作台、回测、进化和 Data Lab。

## 7. 回测

进入 `回测` 标签后，可以发起单次回测任务。

### 7.1 可选参数

- 策略模板：`core-btc-v1` / `core-eth-v1`
- 参数来源：
  - `当前 champion`
  - `指定 challenger`
  - `自定义 JSON`
- 初始资金 USDT
- 初始冷封存资产

如果选择：

- `指定 challenger`：需要从下拉框选择一个 challenger gene
- `自定义 JSON`：需要填写合法的 ParamPack JSON

### 7.2 操作步骤

1. 选择模板
2. 选择参数来源
3. 填写初始资金与冷封存资产
4. 点击 `开始回测`

### 7.3 结果查看

任务创建后，页面会显示：

- 任务状态
- 最终权益
- 总注资
- 收益率
- 最大回撤
- 成交次数
- NAV 曲线

## 8. 进化

进入 `进化` 标签后，可以运行 GA 进化任务。

### 8.1 可选参数

- 策略模板
- Spawn 模式：
  - `继承当前最优`
  - `随机探索`
  - `手动指定`
- 种群大小
- 最大代数

如果选择 `手动指定`，需要填写合法的 SpawnPoint JSON。

### 8.2 操作步骤

1. 选择模板
2. 选择 Spawn 模式
3. 填写种群大小与最大代数
4. 点击 `启动新一轮进化`

### 8.3 结果查看

页面会显示：

- 当前运行任务
- 当前代数
- 最佳评分
- 当前回撤
- 历史任务列表
- 当前 champion
- 基因库

对于 `challenger` 记录，如果它来自某个完成的进化任务，会出现 `晋升为 champion` 按钮。

## 9. Data Lab

进入 `Data Lab` 标签后，可以管理历史 K 线数据。

支持的标的：

- `BTCUSDT`
- `ETHUSDT`

### 9.1 从 Bitget 同步历史 K 线

填写：

- 数据标的
- 同步 K 线数量，例如 `600`

点击 `从 Bitget 同步历史 K 线`。

### 9.2 导入 1h CSV

1. 选择目标标的
2. 选择本地 CSV 文件
3. 点击 `导入 CSV`

说明：

- CSV 导入只面向 `1h` K 线
- 导入后会刷新覆盖范围和最近数据预览

### 9.3 页面展示内容

Data Lab 会显示：

- 各标的覆盖范围
- bar 数量
- 首条与末条时间
- 最近收盘价
- 最近 24 根 K 线走势
- 最近若干条 K 线表格

## 10. 推荐使用顺序

建议按下面顺序使用：

1. 注册账号并登录
2. 在 `Data Lab` 准备 `BTCUSDT` 或 `ETHUSDT` 的 `1h` 历史数据
3. 在 `工作台` 创建实例
4. 先做 `回测`
5. 再做 `进化`
6. 如果要联通实盘，再额外启动 `LocalAgent`

## 11. 常见问题

**1. 登录失败**

- 先确认账号已经通过注册接口创建
- 再确认页面里的 `API Base` 指向正确地址

**2. 页面提示接口错误**

- 检查 SaaS 是否已启动
- 检查 `http://127.0.0.1:18080/healthz`
- 检查数据库和 Redis 是否可用

**3. 回测或进化没有结果**

- 先确认 `Data Lab` 中已经有对应标的的历史 K 线

**4. Agent 显示离线**

- 这表示本地 Agent 还没有启动
- 不影响你先使用 Dashboard、回测、进化和 Data Lab

**5. `lab` 模式下为什么不能创建或启动实例**

- 这是 v1 约束
- `lab` 只负责回测、进化和数据实验，不负责真实实例写操作
